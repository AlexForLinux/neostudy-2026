Оценка качества агентов

Сценарии использования агента

Сценарий 1

Условие: Ответ на вопрос пользователя есть в локальной базе знаний Happy Path:

  1. Агент обратился к текстовому ретриверу.
  2. Агент собрал достаточно данных.

Сценарий 2

Условие: Ответ на вопрос пользователя отсутсвуте в локальной базе знаний Happy path:

  1. Агент обратился к текстовому ретриверу. Понял, что этих данных недостаточно или что нужные данные отустсвуют.
  2. Агент сделал запрос через WebSearch.
  3. Агент выбрал подходящий из результатов запроса источник и прочитал его.
  4. Агент собрал достаточно данных.

Метрики оценки

  1. Среднее число шагов на задачу по всем задачам и по каждому сценарию отдельно
  2. Средняя оценка соотвествия ожидаемому сценарию по всем задачам и по каждому сценарию отдельно
  3. Средняя оценка качества ответа по всем задачам и по каждому сценарию отдельно
  4. Среднее время всего цикла рассуждения по всем задачам и по каждому сценарию отдельно

Способо формирования данных для оценки

  1. Составить 10 запросов пользователей, 5 из которых идут по сценарию 1, а 5 по сценарию 2.
  2. Каждый такой запрос повторить K раз
  3. Занести полученные результаты в таблицу со столбцами:
  • запрос-задача
  • номер сценария
  • число шагов
  • время выполнения
  • использование инструментов (последовательность вызовов X->Y->Z)
  • соотвествие сценарию (2 - полное соответсвие, 1 - частничное соотвествие, 0 - отсутствие соответствия)
  • оценка итогового ответа (0-5, где 0 - агент не достиг цели, ответ не соотвествует запросу, 5 - агент успешно достиг цели, ответ полностью соотвествует запросу)
  1. Рассчитать метрики
